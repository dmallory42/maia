# Maia Framework Design

An opinionated, API-first PHP 8.2+ framework designed for simplicity, security, and agent-friendly workflows.

## Principles

- **Simplicity** — easy to set up, clear syntax, minimal configuration
- **Security by default** — not an afterthought; CORS, rate limiting, security headers, input validation all built in
- **Opinionated** — aimed at API backends for React/JS frontends (but not limited to)
- **Controller -> Service separation** — enforced by convention
- **Agent-friendly** — CLI-first approach; structured output for agentic workflows
- **Lean dependencies** — use external packages for complex/security-critical things (JWT), build simple utilities in-house (logging, config)

## Architecture

Micro-kernel with first-party packages:

```
maia/core    — Router, DI Container, Middleware Pipeline, Request/Response
maia/orm     — Query Builder, Model, Migrations, Schema
maia/auth    — JWT, API Key middleware
maia/cli     — Scaffolding, project management, inspection commands
```

Each package is independent in its Composer definition but ships together as `maia/framework`.

### Conventions

- **Controllers** only handle request -> response. They call Services for logic.
- **Services** contain business logic. Plain classes, auto-wired via constructor.
- **Models** are lightweight active-record classes tied to DB tables.
- **Middleware** implements `handle(Request $request, Closure $next): Response`.
- **`maia.json`** describes project structure so the CLI and agents know where to scaffold.
- **`CLAUDE.md`** and **`AGENTS.md`** are auto-generated on `maia new` describing the app's structure, routes, and conventions.

## Core: Router, Request/Response, Middleware

### Request/Response

```php
// Immutable Request object
$request->method();              // 'GET'
$request->path();                // '/users/42'
$request->param('id');           // '42' (from route)
$request->query('page');         // from query string
$request->body();                // parsed JSON body (auto-decoded)
$request->header('Authorization');
$request->bearerToken();         // extracted Bearer token
$request->user();                // user set by auth middleware
$request->validated();           // validated fields (from FormRequest)

// Fluent Response builder
Response::json(['user' => $data], 200);
Response::error('Not found', 404);
Response::empty(204);
```

### Routing

Attribute-based with a single unified `#[Route]` attribute:

```php
#[Controller('/users')]
class UserController
{
    #[Route('/', method: 'GET')]
    public function list(UserService $service): Response
    {
        return Response::json($service->getAll());
    }

    #[Route('/{id}', method: 'GET')]
    public function show(int $id, UserService $service): Response
    {
        return Response::json($service->find($id));
    }

    #[Route('/', method: 'POST', middleware: [AuthMiddleware::class])]
    public function create(Request $request, UserService $service): Response
    {
        $user = $service->create($request->body());
        return Response::json($user, 201);
    }
}
```

- `method` defaults to `'GET'` if omitted
- `middleware` can be specified inline on the route attribute
- `#[Controller('/prefix')]` sets a path prefix for all routes in the class
- `#[Middleware(...)]` can also be applied at the class level

### Middleware Pipeline

```php
class AuthMiddleware implements Middleware
{
    public function handle(Request $request, Closure $next): Response
    {
        $token = $request->bearerToken();
        if (!$this->jwt->verify($token)) {
            return Response::error('Unauthorized', 401);
        }
        $request = $request->withAttribute('user', $this->jwt->decode($token));
        return $next($request);
    }
}
```

Execution order: global (config/middleware.php) -> controller-level -> route-level.

## Dependency Injection

Auto-wiring from constructor type hints. No Providers, no interface mapping.

```php
class UserService
{
    public function __construct(
        private UserRepository $repo,
        private Logger $logger
    ) {}
}
```

For config-dependent objects, a factory registry in `config/app.php`:

```php
return [
    'factories' => [
        DatabaseConnection::class => fn() => new DatabaseConnection(
            config('database.host'),
            config('database.name'),
        ),
    ],
    'singletons' => [
        DatabaseConnection::class,
    ],
];
```

Container capabilities:
- **Auto-wiring** — resolves concrete classes from type hints, zero config
- **Factories** — callables for objects that need manual construction
- **Singletons** — shared instances (DB connections, etc.)
- **Constructor injection only** — no property injection, no setter injection

## ORM

Lightweight active-record layer over PDO. Database-agnostic via PDO drivers.

### Model Definition

```php
#[Table('users')]
class User extends Model
{
    public int $id;
    public string $name;
    public string $email;
    public DateTime $created_at;

    #[HasMany(Post::class)]
    public array $posts;
}

#[Table('posts')]
class Post extends Model
{
    public int $id;
    public int $user_id;
    public string $title;

    #[BelongsTo(User::class)]
    public User $user;
}
```

### Querying

```php
$user = User::find(42);

$users = User::query()
    ->where('active', true)
    ->where('role', 'admin')
    ->orderBy('created_at', 'desc')
    ->limit(10)
    ->get();

$user = User::query()->where('email', $email)->first();

// Eager loading relationships
$user = User::query()->with('posts')->find(42);
```

### Creating/Updating

```php
$user = User::create(['name' => 'Mal', 'email' => 'mal@example.com']);

$user->name = 'Updated';
$user->save();

User::query()->where('active', false)->update(['archived' => true]);
```

### Relationships

Attribute-based, convention-driven foreign keys:

- `#[BelongsTo(User::class)]` infers `user_id` foreign key
- `#[HasMany(Post::class)]` infers `user_id` on posts table
- Override with: `#[BelongsTo(User::class, foreignKey: 'author_id')]`
- Lazy loaded by default, eager via `->with('relationship')`

### Migrations

```php
return new class extends Migration
{
    public function up(Schema $schema): void
    {
        $schema->create('users', function (Table $table) {
            $table->id();
            $table->string('name');
            $table->string('email')->unique();
            $table->boolean('active')->default(true);
            $table->timestamps();
        });
    }

    public function down(Schema $schema): void
    {
        $schema->drop('users');
    }
};
```

All queries use prepared statements. No raw string interpolation.

## Authentication & Security

### JWT Authentication

```php
// config/auth.php
return [
    'jwt' => [
        'secret' => env('JWT_SECRET'),
        'ttl' => 3600,
        'refresh_ttl' => 86400,
        'algorithm' => 'HS256',
    ],
    'api_keys' => [
        'header' => 'X-API-Key',
        'hash_algorithm' => 'sha256',
    ],
];
```

```php
#[Controller('/admin')]
#[Middleware(Auth::jwt())]
class AdminController
{
    #[Route('/dashboard', method: 'GET')]
    public function dashboard(Request $request): Response
    {
        $user = $request->user();
        return Response::json(['welcome' => $user->name]);
    }
}
```

### API Key Authentication

```php
#[Controller('/webhooks')]
#[Middleware(Auth::apiKey())]
class WebhookController
{
    #[Route('/stripe', method: 'POST')]
    public function stripe(Request $request): Response { }
}
```

### Security Defaults (Global, Out of the Box)

- **CORS** — configured in `config/cors.php`, restrictive by default (no wildcard origins)
- **Rate limiting** — built-in middleware: `#[Route('/login', method: 'POST', middleware: [RateLimit::perMinute(5)])]`
- **Request validation** — type-safe via FormRequest classes
- **Automatic JSON content-type enforcement** — rejects non-JSON bodies on API routes
- **Security headers** — `X-Content-Type-Options`, `X-Frame-Options`, `Strict-Transport-Security`
- **Input sanitization** — all request inputs trimmed and null-converted by default
- **CSRF protection** — disabled by default for API routes (JWT handles this), available for session-based use

### Request Validation

```php
class CreateUserRequest extends FormRequest
{
    public function rules(): array
    {
        return [
            'name' => ['required', 'string', 'max:255'],
            'email' => ['required', 'email', 'unique:users'],
            'password' => ['required', 'string', 'min:8'],
        ];
    }
}
```

## CLI

The `maia` CLI is the primary entry point for developers and agents.

### Commands

```bash
# Project lifecycle
maia new my-app
maia up                          # Start dev server
maia up --port=8080

# Scaffolding
maia create:controller UserController
maia create:service UserService
maia create:model User
maia create:middleware RateLimitMiddleware
maia create:request CreateUserRequest
maia create:migration create_users_table

# Database
maia migrate
maia migrate:rollback
maia migrate:status
maia db:seed

# Inspection (agent-friendly)
maia routes                      # List all registered routes
maia routes --json               # JSON output for agents
maia describe                    # Project structure overview
maia describe --json             # JSON manifest of project state

# Testing
maia test
maia test --filter=UserTest
maia create:test UserServiceTest
```

### Agent-Friendly Design

- All commands support `--json` for structured output
- All commands support `--help` for human-readable docs
- `maia describe` outputs routes, models, middleware stack, config — everything an agent needs
- Error output follows a standard format:

```json
{
    "error": true,
    "code": "MIGRATION_FAILED",
    "message": "Column 'email' already exists in table 'users'",
    "file": "database/migrations/2026_02_25_create_users_table.php",
    "suggestion": "Check if this migration has already been run with 'maia migrate:status'"
}
```

## Error Handling

Centralized exception handler converts exceptions to JSON API responses.

### Built-in Exceptions

```php
throw new NotFoundException('User not found');         // 404
throw new ValidationException($errors);                // 422
throw new UnauthorizedException('Invalid token');      // 401
throw new ForbiddenException('Insufficient role');     // 403
throw new ConflictException('Email already exists');   // 409
```

### Environment-Aware Output

- **Development**: full error details, exception class, file, line, trace
- **Production**: generic error message with HTTP status code only

## Logging

Built-in, minimal, JSON-structured file logger.

```php
// config/logging.php
return [
    'channel' => 'file',          // 'file', 'stderr', 'null'
    'path' => 'storage/logs/maia.log',
    'level' => env('LOG_LEVEL', 'info'),
    'format' => 'json',
];
```

API: `$logger->info()`, `$logger->warning()`, `$logger->error()`, `$logger->debug()`.

No rotation or aggregation — those are ops concerns.

## Testing

PHPUnit integration with built-in helpers.

```php
class UserControllerTest extends TestCase
{
    public function test_list_users(): void
    {
        $response = $this->get('/users');
        $response->assertStatus(200);
        $response->assertJsonStructure([['id', 'name', 'email']]);
    }

    public function test_create_requires_auth(): void
    {
        $response = $this->post('/users', ['name' => 'Test']);
        $response->assertStatus(401);
    }

    public function test_create_with_auth(): void
    {
        $response = $this->withToken($this->generateJwt(['role' => 'admin']))
            ->post('/users', ['name' => 'Test', 'email' => 'test@test.com']);
        $response->assertStatus(201);
    }
}
```

### Test Helpers

- `$this->resolve(Class::class)` — resolve from container with test overrides
- `$this->get()`, `$this->post()`, etc. — HTTP test helpers (no server needed)
- `$this->withToken()` — attach auth for protected routes
- `$this->generateJwt()` — generate test JWT tokens
- Auto-migrated in-memory SQLite for database tests
- `$this->assertDatabaseHas('users', ['email' => '...'])` — DB assertions
